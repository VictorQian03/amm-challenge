## Branch Portfolio

### `local_refine`

- `local-refine-hysteretic-release`
  - Branch type: local refinement
  - Anchor: `apr11-screen480-0948 screen_0003`
  - Hypothesis: keep the continuation-gated passive recapture idea, but replace its linear long-gap release with a hysteretic deadband plus minimum dwell time. Only let passive recapture spend after repeated reconvergence confirms the book is back inside a narrow safe region, and snap back to protection immediately on renewed extension.
  - Why now: this is the narrowest mechanistic refinement of the best raw survivor and directly targets the current plateau where raw gains are real but too small.

### `anti_arb`

- `anti-arb-toxicity-budget`
  - Branch type: anti-arb
  - Hypothesis: add a bounded toxicity budget driven by signed-flow persistence and price-response agreement, then spend it only on the toxic side. This imports VPIN-style flow-toxicity thinking without reviving the failed retail-share classifier, because relief is conditioned on adverse-selection evidence rather than estimated benign share.
  - Why now: the latest broad pivot that improved retail edge still leaked arb; this branch attacks that leak directly.

### `weak_slice`

- `weak-slice-rebate-bucket`
  - Branch type: weak slice
  - Hypothesis: replace broad calm rebates with a token-bucket rebate budget that fills only during benign long-gap, low-hazard periods and drains quickly when used. Weak slices then get occasional concentrated price improvement without leaving the surface permanently cheap in medium-risk regimes.
  - Why now: the weak buckets need cheaper quotes, but every broad calm-band variant has been punished for making that relief too continuous.

### `fee_discipline`

- `fee-discipline-slew-scheduler`
  - Branch type: fee discipline
  - Hypothesis: keep the current signal stack but pass the desired fee through an asymmetric slew-rate scheduler: fast up, slow down, with tighter relaxation caps in calm regimes. The intended effect is to preserve burst defense while removing unnecessary oscillation and fee-jump overshoot that can damage calm competitiveness.
  - Why now: recent regressions show that both flattened surfaces and spiky threshold surfaces fail; disciplined fee motion is a different control axis.

### `structural_pivot`

- `structural-pivot-two-mode-controller`
  - Branch type: structural pivot
  - Hypothesis: replace the single additive quote map with a two-mode controller: one competitive mode and one protective mode, switched by a slow hysteretic router with dwell time. Each mode has a simpler fee map than the current monolith, which is intended to sharpen mode separation without reusing the failed shock-spike or simple-asymmetric families.
  - Why now: the current additive spine appears locally saturated, and this is the clearest orthogonal pivot that is still implementable in one file.
