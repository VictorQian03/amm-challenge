## Branch Portfolio

### `fee_discipline`

- `fee-discipline-vol-accumulator-floor`
  - Branch type: fee discipline
  - Hypothesis: replace part of the current instantaneous event repricing with a filtered volatility accumulator and an exponentially decaying protective floor. Rapid same-regime moves should keep the floor elevated, but quiet gaps should release it gradually rather than via abrupt jumps.
  - Why now: directly attacks the observed `over_spiky_fee_surface` bottleneck without repeating the over-flattening from `prescreen_0008`.

### `local_refine`

- `local-refine-concave-relief`
  - Branch type: local refinement
  - Hypothesis: keep the incumbent signal stack, but change only the nonlinear map: make protection convex only above a real risk threshold, and make calm/opportunity relief concave and capped. The aim is to stop broad cheap quoting while preserving occasional weak-slice competitiveness.
  - Why now: the best raw survivor improved calm slices, but it bought those gains with too much extra average cheapness.

### `anti_arb`

- `anti-arb-side-markout-memory`
  - Branch type: anti-arb
  - Hypothesis: add a side-specific markout memory that only charges the toxic side when trade direction and subsequent price extension agree. This separates symmetric baseline spread from asymmetric informed-flow protection.
  - Why now: prior anti-arb overlays improved arb discipline somewhat, but they still spent too much through the same spiky surface.

### `weak_slice`

- `weak-slice-exp-decay-envelope`
  - Branch type: weak slice
  - Hypothesis: replace inert or broad rebate buckets with a bounded calm-relief envelope that triggers only after repeated benign confirmations, then decays exponentially. That gives weak slices short bursts of competitiveness without leaving the book cheap continuously.
  - Why now: the bucket family was either harmful or inert, which suggests the spend path needs persistence and decay, not a refill/spend toggle.

### `structural_pivot`

- `structural-pivot-defensive-router-v2`
  - Branch type: structural pivot
  - Hypothesis: retry the two-mode idea with a defensive default, firmer protective floor, and much slower release into the competitive mode. The cheap mode should be bounded by the same protective envelope rather than allowed to set the whole surface.
  - Why now: `prescreen_0009` falsified the naive cheap-router design, not the entire notion of topology separation.
